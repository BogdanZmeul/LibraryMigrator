## Швидкий старт:
### 1. Налаштуйте змінні оточення: 
Склонуйте та перейменуйте .env.example на .env та додайті ключі

### 2. Зберіть та запустіть контейнер:
```bash
docker-compose up -d --build
```

Після цього контейнер працюватиме у фоні.

## Основні команди
### Щоб запустити головний скрипт (main.py):
```bash
  docker-compose exec migrator_agent python main.py
```

### Робота з Git всередині Docker
Якщо потрібно перевірити стан репозиторію в sandbox:
```bash
docker-compose exec migrator_agent git -C sandbox status
```

### Робота з внутрішнім репозиторієм
Оскільки папка sandbox є спільною з docker, тому можна її переглядати локально

## Структура Sandbox
- sandbox/ — це ваша "ігрова зона". Вона проігнорована в основному Git.
- Агент закидає сюди код, який потребує оновлення.
- Шлях до проєктів всередині контейнера завжди починається з /app/sandbox/.

## Коли потрібно перезбирати Docker?
Вам не потрібно перезбирати контейнер, якщо ви просто змінюєте код (завдяки Volumes). 

Перезбірка (`docker-compose up -d --build`) потрібна лише якщо ви:

1. Додали нову бібліотеку в requirements.txt.
2. Змінили сам Dockerfile.
3. Змінили системні налаштування в docker-compose.yml.


## Troubleshooting
