## AI Library Migrator
Професійний інструмент для автоматичної міграції застарілих бібліотек за допомогою AI

## Як користуватися (CLI Mode)
Це основний режим для міграції реальних проєктів. Ви запускаєте агента прямо в папці вашого проєкту.
```bash
# 1. Перейдіть у папку вашого проєкту
cd /шлях/до/вашого/проєкту

# 2. Запустіть міграцію, вказавши шлях до скрипта run.sh
# Використовуйте параметри для конкретної бібліотеки та версій
"/шлях/до/агента/run.sh" -l <назва_бібліотеки> -ov <стара_версія> -nv <нова_версія>
```

## Розробка агента (Development Mode)
Якщо ви змінюєте код самого агента, використовуйте Docker Compose для швидкого тестування та налагодження логіки.

### 1. Запуск контейнера у фоні:
```bash
docker-compose up -d
```
Завдяки Volumes зміни в коді агента підхоплюються миттєво без перезбірки.

### 2. Використання Sandbox:
Папка sandbox/ всередині агента призначена для локальних тестів. Просто скопіюйте туди будь-який репозиторій для тестів.
```bash
docker-compose exec migrator_agent python main.py /app/sandbox/<Назва репозиторія> -l <назва_бібліотеки> -ov <стара_версія> -nv <нова_версія>
```

### 3. Перевірка встановлених пакетів:
```bash
docker-compose exec migrator_agent pip list
```

## Коли перезбирати Docker?
Вам потрібно запустити `docker-compose build`, лише якщо:
1. Змінився `requirements.txt` (нові залежності).
2. Змінився `Dockerfile`.
3. Ви хочете "запекти" останню версію коду в образ для стабільного використання через `run.sh`.

## Важливо для Windows
Якщо в шляху до агента є пробіли, завжди беріть шлях до `run.sh` у лапки:
```bash
"C:\My Projects\LibraryMigrator\run.sh" -l numpy -ov 1.2 -nv 1.25
```
